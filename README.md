# BLNDR+SKTCHP-sync

**Save-based sync of geometry and materials between SketchUp and Blender.**

Синхронизация геометрии и материалов между SketchUp и Blender **по сохранению файла**.
Работаете по очереди: подвинули в Blender → Ctrl+S → в SketchUp объект уже обновился.
Размеры и сцены живут в SketchUp, свет и рендер — в Blender; синк их не трогает.

## Как это работает

Рядом с файлами проекта создаётся папка-мост `.su_blender_sync`:

```
МойПроект/
  дом.skp
  дом.blend          ← оба файла в одной папке (это и есть привязка)
  .su_blender_sync/
    from_sketchup.glb + manifest_sketchup.json   ← пишет SketchUp
    from_blender.glb  + manifest_blender.json    ← пишет Blender
```

Каждая сторона при сохранении экспортирует свою синхронизируемую геометрию в GLB
и манифест с номером ревизии каждого объекта. Вторая сторона проверяет мост раз в
2 секунды и импортирует **только объекты с выросшей ревизией** — поэтому:

- нет эхо-циклов (неизменённые объекты сохраняют ревизию);
- размеры в SketchUp на неизменённых объектах не отвязываются;
- программы не обязаны быть запущены одновременно.

При импорте в SketchUp выполняется автоочистка: слияние копланарных треугольников,
сглаживание рёбер кривых поверхностей (< 30°), сжатие текстур до 1024 px.

Детали протокола — в [docs/PROTOCOL.md](docs/PROTOCOL.md).

## Установка

**SketchUp (2025+, нужен нативный glTF):** скопировать содержимое `sketchup/`
в папку Plugins (`%AppData%\SketchUp\SketchUp 2025\SketchUp\Plugins`), либо запустить
`tools\deploy_sketchup.cmd`. Перезапустить SketchUp. Меню: `Extensions → SU ⇄ Blender Sync`.

**Blender (4.0+):** `Edit → Preferences → Add-ons → Install from Disk...` →
`blender/blender_su_sync.py`, включить. Панель: `Properties → Scene → SketchUp Sync`.

**Синк по умолчанию выключен.** Включение: SketchUp — `Extensions → SU ⇄ Blender
Sync → Enabled` (запоминается между сессиями); Blender — галочка **Sync on save**
в панели (хранится в сцене). Срабатывает только на явное сохранение —
авто-сохранение SketchUp отфильтровывается.

## Правила

1. `.skp` и `.blend` — в одной папке.
2. SketchUp: синхронизируются группы/компоненты **верхнего уровня**; свободную
   геометрию — группировать. Безымянные группы получают автоимя при сохранении.
3. Blender: синхронизируется коллекция **"SketchUp Sync"** (создаётся сама).
   Вложенные в неё коллекции приезжают в SketchUp **группами** с **Tag** имени
   коллекции; в обратную сторону эти группы возвращаются в свои коллекции.
4. Имя объекта = идентичность. Переименование = удалить + создать заново.
5. Per-object побеждает последнее сохранение.

## Структура репозитория

```
sketchup/   расширение SketchUp (Ruby)
blender/    аддон Blender (Python)
tools/      deploy_sketchup.cmd — копирование в Plugins
docs/       протокол моста, заметки по исследованию
```

## Статус: прототип

Известные ограничения: дублирование материалов при повторных импортах;
undo применённого синка трактуется как локальное изменение; модификаторы Blender
применяются при экспорте. Точки первой проверки — в docs/PROTOCOL.md, раздел «Тест-план».
